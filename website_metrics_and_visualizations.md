# 网站展示指标与可视化方案

基于对话总结和数据分析，以下是适合在网站上展示的 AI 使用习惯指标和可视化设计。

## 📊 核心展示指标（Dashboard Cards）

### 1. 总体使用概览
- **总对话数**: 800
- **总消息数**: 13,146
- **使用天数**: 300 天
- **日均对话数**: 3.4
- **日均消息数**: 53.2

**可视化**: 大号数字卡片，带趋势箭头（与上月对比）

---

### 2. 使用模式分类

**对话类型分布**（基于总结分析）:
- 🔧 **深度技术咨询**: 40% (320 对话)
- 📝 **商务文档优化**: 25% (200 对话)
- 🎨 **创意设计协作**: 15% (120 对话)
- 📚 **专业知识学习**: 15% (120 对话)
- 💬 **日常实用咨询**: 5% (40 对话)

**可视化**: 
- 饼图/环形图（带百分比标签）
- 堆叠柱状图（按时间趋势）

---

### 3. 技术能力使用

**代码相关**:
- 包含代码的对话: ~15% (120+ 对话)
- 主要语言: R, Python, SQL
- 代码消息占比: 7.9%

**多模态使用**:
- 包含图片的对话: ~12% (95+ 对话)
- 多模态消息: 2.4%
- 图片分析 + 创意生成

**工具调用**:
- 使用工具的对话: ~30% (240+ 对话)
- Tool 消息占比: 15.6%

**可视化**:
- 三列卡片展示（代码/图片/工具）
- 雷达图（技术能力维度）
- 时间序列堆叠面积图（各技术使用趋势）

---

### 4. 交互模式分析

**对话深度**:
- 平均对话长度: 16.4 条消息
- 最深对话: 226 条消息
- 复杂对话（>20条）: ~25%

**交互类型**:
- 协作型: 40% (与AI共同解决问题)
- 指导型: 35% (寻求技术指导)
- 问答型: 25% (快速查询)

**可视化**:
- 对话长度分布直方图
- 交互模式饼图
- 对话深度 vs 复杂度散点图

---

### 5. 时间使用模式

**活跃时段**:
- 最活跃: 18:00 (晚上)
- 最不活跃: 11:00 (上午)
- 工作日 vs 周末分布

**使用趋势**:
- 早期: 基础咨询为主
- 中期: 深度技术咨询
- 近期: 创意设计 + 综合项目

**可视化**:
- 热力图（星期 × 小时）
- 时间序列折线图（对话数/消息数趋势）
- 日历热力图（每日活跃度）

## 📝 指标计算公式

### 技术深度指数
```
技术深度 = (代码对话数 × 0.4 + 工具使用对话数 × 0.3 + 复杂对话数 × 0.3) / 总对话数 × 100
```

### 创意探索指数
```
创意探索 = (创意对话数 × 0.5 + 图片对话数 × 0.3 + 创新应用对话数 × 0.2) / 总对话数 × 100
```

### 工作流整合度
```
整合度 = (多步骤对话数 + 工具集成对话数) / 总对话数 × 100
```

### 迭代优化倾向
```
优化倾向 = 多次修改对话数 / 总对话数 × 100
```
